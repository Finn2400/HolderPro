# Licensing and attribution

This document explains the project's distribution policy. It is not legal
advice. HolderPro requires independent license review before its first public
binary and again for material changes to static linking or store distribution.

## HolderPro license

The complete shipping repository is licensed under AGPL-3.0-or-later. The
license text is in `LICENSE`. The application must provide a visible
**Help → About / Legal Notices** action containing:

- HolderPro copyright and AGPL terms;
- the no-warranty notice;
- a source link for the exact running version;
- PrusaSlicer attribution, version, and commit;
- third-party notices; and
- the non-affiliation statement below.

## PrusaSlicer relationship

The approved description is:

> HolderPro uses the unmodified PrusaSlicer 2.9.6 Organic-support implementation through a headless adapter.

The source is pinned to PrusaSlicer tag `version_2.9.6`, commit
`b028299c770b8380ee81c921a2867d522f288123`. HolderPro preserves upstream
copyright and license notices and publishes the complete pinned source with
every binary release.

The required relationship statement is:

> HolderPro is an independent project; not affiliated with or endorsed by Prusa Research.

Do not use Prusa logos or presentation that suggests an official Prusa product.

## Generated output

An STL generated by HolderPro is not automatically AGPL merely because
HolderPro produced it. AGPL section 2 says output is covered only when its
content constitutes a covered work. Ordinary geometry derived from the user's
model is not program source code merely because HolderPro processed it. Users
remain responsible for rights in their input and output.

## Binary obligations

Every conveyed binary must be accompanied by convenient access to the matching
corresponding-source archive described in
[corresponding-source.md](corresponding-source.md). Release notes must link the
archive directly. A moving `main` branch link is not sufficient.

Qt libraries are kept dynamically replaceable in one-directory desktop
packages. Their notices, license terms, source link/offer, and relinking rights
must be validated for each resolved release. The SBOM and notice generation
gate catches dependency changes but does not replace legal review.

## Distribution channels

Initial releases are direct GitHub downloads and PyPI wheels. App-store terms
can impose restrictions that conflict with copyleft rights; no Apple App Store,
Microsoft Store, Snap Store, or other restricted store listing is authorized
until a separate AGPL compatibility review approves that channel.

## Trademark gate

Before adopting a logo, signing identity, domain strategy, or stable `1.0`
branding, commission a professional trademark knockout search for HolderPro in
the relevant countries and software/manufacturing classes. Record the written
disposition in the private release records. A web search or package-index check
does not satisfy this gate. The technical evidence and preliminary screening
are organized in [the legal review brief](legal-review-brief.md).
